gcdyn.encode.encode_tree(intree, max_leaf_count=None, ladderize=True, dont_scale=False)[source]#

Returns the “Compact Bijective Ladderized Vector” form of the given tree. Does not modify input tree.

The CBLV has been adapted to include the x attribute of every node. Thus, in reference to figure 2a (v) in Voznica et. al (2022), two additional rows have been appended: a third row of x for the nodes in row 1, and a fourth row of x for the nodes in row 2.

Note that by default this ladderizes the tree first, so if you’ve already done this you should set ladderize to False.

If max_leaf_count is not set, it defaults to the number of leaves in the tree. It often makes more sense to pad encoded trees to the same size right before passing into the model (using encode.pad_trees()) than to try to guess a max_leaf_count when initially encoding them.

Return type:

ndarray[float]
