gcdyn.model.BDMSModel¶

class gcdyn.model.BDMSModel(trees, birth_rate, death_rate, mutation_rate, mutator, sampling_probability, **opt_kwargs)[source]¶

Bases: object

A class that represents a GC model.

Parameters:

  • trees (list[TreeNode]) – list of trees

  • birth_rate (Response) – Initial birth rate response function. If birth_rate.grad == True, will be optimizable via fit().

  • death_rate (Response) – Initial death rate response function. If death_rate.grad == True, will be optimizable via fit().

  • mutation_rate (Response) – Initial mutation rate response function. If mutation_rate.grad == True, will be optimizable via fit().

  • mutator (Mutator) – Generator of mutation effects.

  • sampling_probability (float) – Probability of sampling a survivor.

  • opt_kwargs – keyword arguments to pass to jaxopt.ScipyBoundedMinimize

Given a collection of ete3.TreeNode, fit the parameters of the model.

Parameters:

  • birth_rate_lower_bound (array-like) – lower bounds for the birth rate parameters

  • birth_rate_upper_bound (array-like) – upper bounds for the birth rate parameters

  • death_rate_lower_bound (array-like) – lower bounds for the death rate parameters

  • death_rate_upper_bound (array-like) – upper bounds for the death rate parameters

  • mutation_rate_lower_bound (array-like) – lower bounds for the mutation rate parameters

  • mutation_rate_upper_bound (array-like) – upper bounds for the mutation rate parameters

All array-like arguments should specify the order of the parameters to match the lexographical order of the parameter names (eg. xscale, xshift, yscale, yshift).
