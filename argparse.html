Infer affinity response function on gcdyn simulation using deep learning neural networks. Example usage:

gcd-dl-infer –indir <input dir> –outdir <output dir> –params-to-predict xscale xshift yscale

usage: gcd-dl-infer [-h] --indir INDIR --outdir OUTDIR [--epochs EPOCHS]
                    [--bundle-size BUNDLE_SIZE] [--dropout-rate DROPOUT_RATE]
                    [--learning-rate LEARNING_RATE]
                    [--ema-momentum EMA_MOMENTUM] [--train-frac TRAIN_FRAC]
                    [--validation-split VALIDATION_SPLIT]
                    [--params-to-predict {xscale,xshift,yscale,total_branch_length} [{xscale,xshift,yscale,total_branch_length} ...]]
                    [--test] [--random-seed RANDOM_SEED] [--overwrite]
                    [--use-trivial-encoding] [--dont-scale-params]

Named Arguments#

--indir

input directory with gcdyn simulation output (uses encoded trees .npy, summary stats .csv, and response .pkl files)

--outdir

output directory

--epochs

Default: 30

--bundle-size

Default: 50

--dropout-rate

Default: 0

--learning-rate

Default: 0.001

--ema-momentum

Default: 0.99

--train-frac

train on this fraction of the trees

Default: 0.8

--validation-split

fraction of training sample to tell keras to hold out for validation during training

Default: 0.1

--params-to-predict

Possible choices: xscale, xshift, yscale, total_branch_length

Default: [‘xscale’, ‘xshift’]

--test

sets things to be super fast, so not useful for real inference, but just to check if things are running properly

Default: False

--random-seed

random seed

Default: 0

--overwrite

Default: False

--use-trivial-encoding

Default: False

--dont-scale-params

Default: False

Simulate B cell trees in germinal centers using the birth-death-mutation model. Example usage that samples parameter values from within ranges:

gcd-simulate –debug 1 –outdir <outdir> –xscale-range 0.5 5 –xshift-range -0.5 3 –yscale-range 1 50 –initial-birth-rate-range 2 10 –carry-cap 150 –time-to-sampling-values 10 –n-trials 1

Example usage with multiple subprocesses:

gcd-simulate –debug 1 –outdir <outdir> –carry-cap 150 –time-to-sampling-values 10 –n-trials 100 –n-sub-procs 10

usage: gcd-simulate [-h] [--n-seqs N_SEQS] [--n-trials N_TRIALS]
                    [--n-max-tries N_MAX_TRIES]
                    [--time-to-sampling-values TIME_TO_SAMPLING_VALUES [TIME_TO_SAMPLING_VALUES ...]]
                    [--time-to-sampling-range TIME_TO_SAMPLING_RANGE [TIME_TO_SAMPLING_RANGE ...]]
                    [--simu-bundle-size SIMU_BUNDLE_SIZE]
                    [--min-survivors MIN_SURVIVORS] [--carry-cap CARRY_CAP]
                    [--capacity-method {birth,death,hard,None}] [--seed SEED]
                    [--outdir OUTDIR]
                    [--birth-response {constant,soft-relu,sigmoid}]
                    [--death-value DEATH_VALUE]
                    [--xscale-values XSCALE_VALUES [XSCALE_VALUES ...]]
                    [--xshift-values XSHIFT_VALUES [XSHIFT_VALUES ...]]
                    [--yscale-values YSCALE_VALUES [YSCALE_VALUES ...]]
                    [--xscale-range XSCALE_RANGE [XSCALE_RANGE ...]]
                    [--xshift-range XSHIFT_RANGE [XSHIFT_RANGE ...]]
                    [--yscale-range YSCALE_RANGE [YSCALE_RANGE ...]]
                    [--initial-birth-rate-range INITIAL_BIRTH_RATE_RANGE [INITIAL_BIRTH_RATE_RANGE ...]]
                    [--yshift YSHIFT]
                    [--mutability-multiplier MUTABILITY_MULTIPLIER]
                    [--n-sub-procs N_SUB_PROCS] [--n-max-procs N_MAX_PROCS]
                    [--itrial-start ITRIAL_START] [--debug DEBUG]
                    [--overwrite] [--dont-run-new-simu] [--test]
                    [--make-plots] [--n-to-plot N_TO_PLOT]

Named Arguments#

--n-seqs

Number of sequences to observe

Default: 70

--n-trials

Number of trials/GCs to simulate

Default: 51

--n-max-tries

Number of times to retry simulation if it fails due to reaching either the min or max number of leaves.

Default: 10

--time-to-sampling-values

List of values from which to choose for time to sampling.

Default: [20]

--time-to-sampling-range

Pair of values (min/max) between which to choose at uniform random the time to sampling for each tree. Overrides –time-to-sampling-values.

--simu-bundle-size

By default, we choose a new set of parameters for each tree. If this arg is set, once we’ve chosen a set of parameter values, we simulate this many trees with those values.

Default: 1

--min-survivors

Default: 100

--carry-cap

Default: 300

--capacity-method

Possible choices: birth, death, hard, None

see bdms.evolve() docs. Note that ‘death’ often involves a ton of churn, which makes for very slow simulations.

Default: “birth”

--seed

random seed

Default: 0

--outdir

Default: “/home/runner/work/gcdyn/gcdyn/docs”

--birth-response

Possible choices: constant, soft-relu, sigmoid

birth rate response function

Default: “sigmoid”

--death-value

value (parameter) for constant death response

Default: 0.1

--xscale-values

list of birth response xscale parameter values from which to choose

Default: [2]

--xshift-values

list of birth response xshift parameter values from which to choose

Default: [-2.5]

--yscale-values

list of birth response yscale parameter values from which to choose

Default: [1]

--xscale-range

Pair of values (min/max) between which to choose at uniform random the birth response xscale parameter for each tree. Overrides –xscale-values. Suggest 0.5 5

--xshift-range

Pair of values (min/max) between which to choose at uniform random the birth response xshift parameter for each tree. Overrides –xshift-values. Suggest -0.5 3

--yscale-range

Pair of values (min/max) between which to choose at uniform random the birth response yscale parameter for each tree. Overrides –yscale-values. Suggest 1 50

--initial-birth-rate-range

Pair of values (min/max) for initial/default/average growth rate (i.e. when affinity/x=0). Used to set –yscale.

Default: [2, 10]

--yshift

atm this shouldn’t (need to, at least) be changed

Default: 0

--mutability-multiplier

Default: 0.68

--n-sub-procs

If set, the –n-trials are split among this many sub processes (which are recursively run with this script). Note that in terms of random seeds, results will not be identical with/without –n-sub-procs set (since there’s no way to synchronize seeds partway through))

--n-max-procs

If set (and –n-sub-procs is set), only run this many sub procs at a time (e.g. to conserve memory).

--itrial-start

if running sub procs (–n-sub-procs) set this so each sub proc’s trial index starts at the proper value

Default: 0

--debug

Verbosity level; set to 1 or 2 for more debug output.

Default: 0

--overwrite

Default: False

--dont-run-new-simu

by default, if some trees are already there but others are missing, we try to rerun the missing ones; if this is set we instead ignore any missing ones and just merge any that are there

Default: False

--test

sets some default parameter values that run quickly and successfully, i.e. useful for quick tests

Default: False

--make-plots

Default: False

--n-to-plot

number of tree slice plots to make

Default: 10
